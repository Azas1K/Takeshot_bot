# Деплой бота на Oracle Cloud (бесплатно, 24/7)

Пошагово. Займёт ~30–40 минут в первый раз. В конце бот будет работать
круглосуточно и сам перезапускаться после сбоев/перезагрузок.

---

## Шаг 1. Регистрация в Oracle Cloud
1. Зайди на **https://www.oracle.com/cloud/free/** → «Start for free».
2. Заполни данные, подтверди почту и телефон.
3. Привяжи зарубежную карту — это **только верификация личности**. На тарифе
   Always Free списаний нет (могут временно захолдировать ~1$ и вернуть).
4. Выбери **Home Region** поближе (например Frankfurt / Amsterdam). Регион потом
   не поменять — но для бота это некритично.

---

## Шаг 2. Создать бесплатный сервер (VM)
1. В консоли: гамбургер-меню ☰ → **Compute → Instances → Create instance**.
2. **Image and shape**:
   - Image: **Canonical Ubuntu 24.04** (или 22.04).
   - Shape → Change shape → вкладка **Specialty and previous generation** →
     **VM.Standard.E2.1.Micro** (AMD, x86, помечен «Always Free-eligible»,
     1 ГБ RAM). Для нашего бота этого с запасом.
     > Если хочешь мощнее — вкладка Ampere, `VM.Standard.A1.Flex`, поставь
     > 1 OCPU / 6 GB (тоже в пределах бесплатного). Но E2.1.Micro проще и
     > стабильнее по доступности.
3. **Add SSH keys**:
   - Выбери **Generate a key pair for me** и **скачай приватный ключ**
     (файл `.key`/`.pem`) — без него не зайдёшь на сервер. Сохрани надёжно.
   - (Или загрузи свой публичный ключ, если уже есть.)
4. Networking оставь по умолчанию (создастся VCN, порт SSH 22 откроется сам).
   Входящие порты для бота НЕ нужны — он сам стучится в Telegram (long polling).
5. Нажми **Create**. Через минуту у инстанса появится **Public IP** — запиши его.

---

## Шаг 3. Подключиться по SSH
На своём компьютере (macOS/Linux — Терминал):

```bash
# дать правильные права ключу (один раз)
chmod 600 ~/Downloads/ssh-key-*.key

# подключиться (подставь путь к ключу и IP сервера)
ssh -i ~/Downloads/ssh-key-*.key ubuntu@ВАШ_PUBLIC_IP
```

Пользователь по умолчанию в Ubuntu-образе — **ubuntu**. На вопрос про
доверие ключу ответь `yes`.

---

## Шаг 4. Поставить окружение на сервере
Уже внутри сервера (в SSH-сессии):

```bash
sudo apt update
sudo apt install -y python3-venv python3-pip
```

---

## Шаг 5. Залить файлы бота на сервер
Открой **второй** терминал на своём компьютере (локальный, не SSH) и скопируй
папку проекта на сервер через scp:

```bash
scp -i ~/Downloads/ssh-key-*.key -r "/Users/gleb/Desktop/takeshot_bot" ubuntu@ВАШ_PUBLIC_IP:/home/ubuntu/
```

> Папка `venv` внутри создана под macOS и на сервере не нужна — она
> пересоздастся в следующем шаге. Можно перед заливкой её удалить, чтобы
> копировалось быстрее.

---

## Шаг 6. Установить зависимости и проверить .env
Вернись в SSH-сессию (терминал с сервером):

```bash
cd /home/ubuntu/takeshot_bot
rm -rf venv                     # если скопировалась старая macOS-версия
python3 -m venv venv
./venv/bin/pip install -r requirements.txt

# проверь, что .env заполнен (BOT_TOKEN и ADMIN_GROUP_ID)
cat .env

# тестовый запуск — должно написать "Бот @... запущен"
./venv/bin/python bot.py
```

Напиши боту в Telegram — заявка должна появиться в группе темой. Если всё ок —
нажми `Ctrl+C`, чтобы остановить тестовый запуск, и настроим автозапуск.

---

## Шаг 7. Автозапуск через systemd (работа 24/7)
Файл сервиса уже есть в проекте — `takeshot-bot.service`. Установи его:

```bash
sudo cp /home/ubuntu/takeshot_bot/takeshot-bot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now takeshot-bot
```

Проверка и логи:

```bash
systemctl status takeshot-bot        # должно быть active (running)
journalctl -u takeshot-bot -f        # живые логи, выход — Ctrl+C
```

Теперь бот работает постоянно: перезапустится сам при падении и после
перезагрузки сервера. Компьютер можно выключать.

---

## Обновление кода в будущем
```bash
# залить обновлённые файлы (с локального компа)
scp -i ключ -r "/Users/gleb/Desktop/takeshot_bot"/*.py ubuntu@IP:/home/ubuntu/takeshot_bot/
# перезапустить сервис (на сервере)
sudo systemctl restart takeshot-bot
```

## Важные мелочи
- **Не теряй приватный SSH-ключ** — без него на сервер не попасть.
- **`bot.db` живёт на сервере** в папке бота и переживает перезапуски — не удаляй.
- **Oracle останавливает простаивающие Always Free ARM-инстансы.** Наш бот
  постоянно активен (держит соединение), так что под правило простоя не попадает.
  Для E2.1.Micro (AMD) это правило не применяется.
- **Токен в `.env`** — не выкладывай файл никуда публично.
