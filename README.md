# Take Shot — Telegram-бот для заявок

Бот принимает заявки с сайта [takeshotbasket.ru](https://takeshotbasket.ru) и
превращает переписку в удобную систему для менеджера.

**Как это выглядит для клиента:**
на сайте он жмёт кнопку → открывается бот → видит приветствие «Йоу! Хотите
заиметь экипу ТШ…» → пишет, какой товар хочет (или кидает скриншот) → получает
автоответ «Спасибо за обращение…».

**Как это выглядит для менеджера:**
все заявки падают в одну группу, но **каждый клиент = отдельная Тема (Topic)**.
Менеджер просто пишет ответ прямо в теме нужного клиента — бот доставляет его
клиенту от имени бота. Диалоги не путаются, даже если заявок много.

```
  Сайт (кнопка)  ──►  Бот  ──►  Группа менеджеров
                                  ├── 🧵 Клиент А   ← отвечаешь здесь → клиенту А
                                  ├── 🧵 Клиент Б   ← отвечаешь здесь → клиенту Б
                                  └── 🧵 Клиент В   ← ...
```

## Что нужно сделать один раз (по шагам)

### 1. Создать бота
1. Открой [@BotFather](https://t.me/BotFather) → `/newbot` → задай имя и username.
2. Скопируй токен и вставь в файл `.env` в строку `BOT_TOKEN=` (токен уже вписан,
   если бот создавали ранее).
3. Там же у BotFather: `/mybots` → выбери бота → **Bot Settings → Group Privacy →
   Turn OFF**. Это обязательно, иначе бот не увидит ответы менеджеров в группе.

### 2. Создать группу заявок с темами
1. В Telegram создай **новую группу** (например «ТШ · Заявки»), добавь туда бота.
2. Сделай бота **администратором** группы. Обязательные права:
   **Управление темами (Manage Topics)** и **Отправка сообщений**.
3. Открой настройки группы → включи **Темы (Topics)**.
   (Если пункта нет — группу нужно сделать супергруппой: обычно это происходит
   автоматически при включении тем.)

### 3. Узнать ID группы и вписать в `.env`
Самый простой способ:
1. Временно добавь в группу бота [@getmyid_bot](https://t.me/getmyid_bot) или
   [@RawDataBot](https://t.me/RawDataBot) — он покажет ID чата.
2. ID группы с темами выглядит как `-1001234567890` (начинается с `-100`).
3. Впиши его в `.env`: `ADMIN_GROUP_ID=-1001234567890`, потом удали служебный бот.

> Альтернатива: запусти нашего бота, напиши что-нибудь в группе — в логах в
> терминале будет виден `chat id`.

### 4. Прикрепить бота к кнопке на сайте
Ссылка на бота: `https://t.me/ИМЯ_БОТА` (username из шага 1, без `@`).

В HTML кнопка выглядит так:
```html
<a href="https://t.me/ИМЯ_БОТА" target="_blank" rel="noopener">
  Хочу экипу ТШ
</a>
```
Просто замени старую ссылку (которая вела на личный аккаунт менеджера) на эту.

> Можно добавить метку источника: `https://t.me/ИМЯ_БОТА?start=site` — тогда в
> боте придёт параметр `site`, полезно для аналитики (в базовой версии не
> обрабатывается, но легко добавить).

## Запуск локально (для теста)

```bash
cd takeshot_bot

# создать окружение и поставить зависимости (venv уже создан в папке)
python3 -m venv venv
source venv/bin/activate            # Windows: venv\Scripts\activate
pip install -r requirements.txt

# заполнить .env (BOT_TOKEN и ADMIN_GROUP_ID) — см. .env.example
python bot.py
```

Пока терминал открыт — бот работает. Закрыл терминал / выключил комп — бот
выключился. Для постоянной работы 24/7 нужен сервер (см. ниже).

---

## Деплой на сервер (когда будет готов)

### Вариант A — обычный Linux-сервер (VPS) через systemd
1. Скопируй папку проекта на сервер, поставь зависимости в venv.
2. Создай сервис `/etc/systemd/system/takeshot-bot.service`:

```ini
[Unit]
Description=Take Shot Telegram Bot
After=network.target

[Service]
WorkingDirectory=/opt/takeshot_bot
ExecStart=/opt/takeshot_bot/venv/bin/python bot.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

3. Запусти:
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now takeshot-bot
sudo systemctl status takeshot-bot     # проверить; journalctl -u takeshot-bot -f — логи
```

### Вариант B — Docker
`Dockerfile` (создать при необходимости):
```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "bot.py"]
```
Запуск: `docker build -t takeshot-bot . && docker run -d --env-file .env --restart always -v $PWD/bot.db:/app/bot.db takeshot-bot`

> Важно: файл `bot.db` (связки клиент↔тема) нужно сохранять между перезапусками —
> в примере он монтируется через `-v`.

### Российские PaaS без своего сервера
Если своего VPS нет, подойдут **Amvera Cloud** (amvera.ru) или **Railway** —
заливаешь файлы, прописываешь переменные `BOT_TOKEN` и `ADMIN_GROUP_ID`, команда
запуска `python bot.py`. Важно: процесс должен работать постоянно (не
«serverless» с засыпанием), т.к. бот держит long-polling соединение с Telegram.

## Полезные команды в группе
- Напиши что угодно **в теме клиента** → уйдёт клиенту.
- `/close` в теме → закрыть диалог и отвязать тему. Если клиент напишет снова —
  заведётся новая тема.

## Частые вопросы
- **Менеджер написал в теме, а клиент не получил.** Проверь, что у бота выключен
  Group Privacy (шаг 1.3) и он админ с правом «Управление темами».
- **Бот не создаёт темы.** В группе должны быть включены Темы, а бот — админ с
  правом Manage Topics.
- **Хочу несколько менеджеров.** Просто добавь их в группу — отвечать в темах
  сможет любой участник. Ничего в коде менять не нужно.
- **Отвечать анонимно.** Клиент видит ответы как сообщения от бота, имя
  менеджера не показывается.
